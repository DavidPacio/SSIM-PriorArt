Notes

The Dimension{.Dimension Member} References in the third column above are used in formats and imports if a dimension is involved,
by appending them to a Bro name after a comma e.g. PL.Revenue.GeoSeg.ByDestination,Countries.UK

The first part of the reference is the Dimension short name. The 2nd part, if included, is the normalised, short version of the Dimension Member Taxonomy Name.
Entries with no {.Dimension Member} component to their references are defaults, where just the Dimension short name gives the default. The actual default name is shown in []s.

Dimension Member Type Codes, if present, mean:

DDefault in Taxonomy
BBraiins dimension (member)
M#If a Dim includes M# Dimension Members and that Dim is one of the Allowable Dims of a Bro, inclusion of one of the M# members is Mandatory when posting to that Bro, except in the special case of Dim 29 Officers when an M# choice can be omitted if the Bro's DataType is Money and the Bro excludes DiMe 423 CoSec. This exception allows for aggregate postings for Officers e.g. for Directors Remuneration.
There are 6 types of M# DiMe Types which define whether a Property Dimension is also required, or is optional, and if applicable, the allowable choices, as follows:
M0No property type DiMe is required. M0 types are used with Dim 9 IFAClasses and DiMe 423 Cosec.
M1A Property DiMe from the next Dim is required if that Dim is an Allowed Dim of the Bro. M1 types are used with Dim 12 FAIHoldings and Dim 34 TPAType. For a particular Bro the M1 need for a Property DiMe from the next Dim might be negated by its Hypercube excluding the Dim, as Hypercube 25 does for FAI Loans (no Dim 13 FAITypes), or by adding the Property Dim (13 FAITypes or 35 TPAStatus as appropriate) to the Bro's Excl Dims list.
M2A Property DiMe is required with the choice restricted to: TFAOwnership.OwnedOrFreehold, ShortLeaseholdProperties, and LongLeaseholdProperties (185, 187, 188). M2 types are used only with Dim 10 TFAClasses. The M2 list comprises only Dim 11 TFAOwnership DiMes. Thus Dim 11 must always be an Allowed Dim of a Bro which allows Dim 10. Bros Import ensures this.
M3A Property DiMe is required with the choice restricted to: TFAOwnership.OwnedOrFreehold, Leased (185, 186). As for M2, M3 types are used only with Dim 10 TFAClasses, and the same Dim 11 rule applies.
M4A Property DiMe may optionally be used with an M4 DiMe with the Property DiMe choice restricted to: Officers.Chairman, CEO, ChairmanCEO, CoSecDirector, OfficerType.Executive, OfficerType.Nonexecutive (419, 420, 421, 424, 487, 488). M4 types are used only with Dim 29 Officers. The M4 list includes Property DiMes from two Dims, 29 Oficers and 30 OfficerType. For a particular Bro, the Dim 30 OfficerType members of the M4 list, OfficerType.Executive, OfficerType.Nonexecutive (487, 488), can be excluded by adding Dim 30 OfficerType to the Bro's Excl Dims list.
M5A Property DiMe may optionally be used with an M5 DiMe with the Property DiMe choice restricted to Officers.ChiefPartnerLLP (422). M5 types are used only with Dim 29 Officers. The M5 Property DiMe is also part of Dim 29 Oficers.
